# HT_SC — نظام تسجيل ومراقبة رحلات الجوية وادارة الساحات الخارجية 

نظام طرفية تفاعلي لمعالجة رسائل واتساب الحافلات، وإدارة ملفات الرحلات،
وتوليد التقارير، ومراقبة البيانات آنياً عبر قاعدة بيانات مركزية.

## المميزات

- معالجة رسائل واتساب وتحويلها إلى سجلات منظمة
- مزامنة ثنائية بين ملفات إكسل وقاعدة البيانات
- توليد تقارير تفاعلية ورسوم بيانية
- مراقب آني لرسائل واتساب (وحدة مستقلة بجافاسكربت)
- نظام تسجيل أحداث متقدم (7 جداول مخصصة)
- واجهة قائمة تفاعلية تدعم اللغة العربية

## المتطلبات

- بايثون 3.12 أو أحدث
- نود جي إس 18+ (لوحدة المراقبة)
- نظام ويندوز 11 (مُختبر عليه)
- ويندوز ترمينال (لدعم العربية المتصلة)

## التثبيت

```bash
# استنساخ المشروع
git clone https://github.com/<username>/HT_SC_.git
cd HT_SC_

# إنشاء البيئة الافتراضية وتفعيلها
python -m venv .venv
.venv\Scripts\activate

# تثبيت المتطلبات
pip install -e .

# تثبيت أدوات التطوير (اختياري)
pip install -e ".[dev]"
```

## التشغيل

```bash
# تشغيل النظام
python main.py

# أو عبر ملف الإطلاق
HT_SC_WT.bat
```

## قواعد البيانات

يستخدم النظام 3 قواعد بيانات SQLite:

| القاعدة | الوظيفة |
|---------|---------|
| `db/ht_sc.db` | القاعدة الرئيسية — رحلات، رسائل، مرسلين، مخالفات |
| `db/ht_sc_new.db` | سجلات موسم 2026 — بيانات النوبات والإحصائيات |
| `logs/ht_sc_logs.db` | سجلات النظام — 7 جداول لتتبع الأحداث والأخطاء |

جميع القواعد تعمل بوضع WAL مع مهلة انتظار 5 ثوانٍ للوصول المتزامن.

## أوامر التطوير

```bash
# تشغيل الاختبارات
python -m pytest tests/ -v

# فحص الكود
ruff check scripts/ db/

# تنسيق الكود
ruff format scripts/ db/

# فحص الأنواع
mypy scripts/ db/ --ignore-missing-imports

# تشغيل المراقب
cd monitor_v2 && npm install && node index.js
```
